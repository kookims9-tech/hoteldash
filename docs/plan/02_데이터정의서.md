## 2. 매출 데이터 (`revenue`)

**시트:** `REVENUE_SHEET_ID` · 탭 `revenue`  
**로더:** `lib/sheets.py` → `load_revenue()`

| 컬럼 | 타입 | 설명 | 검증 |
|------|------|------|------|
| `date` | date | 일자 | NOT NULL, YYYY-MM-DD |
| `occupancy_room` | number | 객실 점유율(%) | 0~100 권장 |
| `room_revenue` | number | 객실 매출 | >= 0 |
| `spa_revenue` | number | 온천 매출 | >= 0 |
| `fb_revenue` | number | F&B 매출 | >= 0 |
| `total_revenue` | number | 합계 | room+spa+fb 일치 권장 |

**기간:** 일별 (현재 약 2025-06 ~ 2026-05)

---

| 컬럼 | 타입 | UI 표기 | 비고 |

| `room_no` | string | 객실번호 | |
| `cleaner` | string | 담당자 | |

| `inspector` | string | 검수자 | |
| `inspector_date` | date | 검수일 | |
| `result` | string | 검수결과 | `불량` 등 (`DEFECT_RESULTS`) |
| `repair_staff` | string | 수리 담당 | |

## 4. VOC 데이터 (샘플)

**소스:** `data/voc.py` · **로더:** `lib/metrics.py` → `voc_summary()`

### 4.1 집계 필드

| 필드 | 타입 | 설명 |
|------|------|------|
| `average_rating` | float | 평균 평점 (5점) |
| `total_reviews` | int | 전체 리뷰 수 |
| `complaints_by_category` | array | `{category, count}` |

### 4.2 리뷰 레코드

| 필드 | 타입 | 설명 |
|------|------|------|
| `date` | string | 리뷰 일자 |
| `rating` | int | 1~5 |
| `channel` | string | 네이버·구글·야놀자 등 |
| `text` | string | 리뷰 본문 |

**2차 확장 컬럼 (설계):** `review_id`, `source_platform`, `sentiment`, `room_type`, `stay_date`

## 5. AX 인사이트 파생 데이터

| 입력 | 모듈 | 산출 |
|------|------|------|
| revenue + rooms + voc | `lib/ax_insights.py` | 이상징후 카드 |
| revenue (trailing 12M) | `lib/executive_analysis.py` | 연중·계절·부문·분기 보고 |
| voc reviews | `lib/ai_reviews.py` | 키워드·요약 (OpenAI optional) |

**분석 기간 규칙:**
- Daily Performance: 데이터 **최신 월** 기준
- AX 연간 분석: 최신 월 포함 **역 12개월**

## 6. 데이터 입력 형식

### 6.1 Phase 1 (확정)

| 데이터 | 방식 | 비고 |
|--------|------|------|
| 매출 | Google Sheets CSV Export | 공개 보기 |
| 객실 | Google Sheets CSV Export | 탭 `Rooms` |
| VOC | Mock (`data/voc.py`) | |
| Excel/CSV 업로드 | 미사용 | 발표 백업용만 |
| Mock fallback | **사용 안 함** | 실패 시 화면 중단 |

### 6.2 Phase 2 (DB)

```
Google Sheets → ETL (15분) → Supabase Postgres → API → Frontend
```

- ETL: `revenue_daily`, `room_inspections` upsert
- VOC: OTA API / Sheets / CSV → `voc_reviews`
- **Supabase 권장** (관계형 집계·Vercel API 궁합)
- Firebase: 푸시·모바일 보조용
