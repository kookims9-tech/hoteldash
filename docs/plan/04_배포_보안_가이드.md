## 1. 환경변수

### 1.1 변수 목록

| 변수 | 필수 | 기본값 | 용도 |
|------|------|--------|------|
| `REVENUE_SHEET_ID` | O | — | 실적 시트 (`.env` / Cloud Secrets) |
| `ROOMS_SHEET_ID` | O | — | 객실 시트 (`.env` / Cloud Secrets) |
| `HOTEL_NAME` | O | ○○온천관광호텔 | UI 표시 |
| `REFRESH_SECONDS` | X | 60 | `@st.cache_data` TTL |
| `OPENAI_API_KEY` | X | — | AI 리뷰 요약 (optional) |
| `OPENAI_MODEL` | X | gpt-4o-mini | OpenAI 모델 |

### 1.2 로컬 개발

프로젝트 루트에 `.env` 생성 (`.env.example` 참고). `.env`는 git에 포함하지 않음.

```bash
cp .env.example .env
# .env 편집 후
source VENV/bin/activate
pip install -r requirements.txt
streamlit run app.py
```

### 1.3 Streamlit Cloud Secrets

[.streamlit/secrets.toml.example](../../.streamlit/secrets.toml.example) 형식을 Cloud 대시보드 **Secrets**에 붙여넣기.

`config.py`는 `os.getenv()`로 읽으며, Streamlit Cloud는 secrets를 환경변수로 주입합니다.

## 2. Streamlit Community Cloud 배포 체크리스트

### 2.1 사전 준비

- [ ] GitHub 저장소에 코드 push (main 브랜치)
- [ ] `requirements.txt`에 `altair`, `requests`, `certifi` 포함 확인
- [ ] `.env`, `VENV/`, `credentials.json` 미포함 확인
- [ ] Google Sheets 2개 **링크가 있는 사용자 보기** 설정
- [ ] 탭 이름: `revenue`, `Rooms` (대소문자 일치)

### 2.2 배포 단계

1. https://share.streamlit.io 접속 → GitHub 연동
2. **Repository** 선택, **Branch** `main`
3. **Main file path:** `app.py`
4. **Advanced settings → Secrets:** `secrets.toml.example` 내용 입력 (실제 ID)
5. **Deploy** → `https://<app-name>.streamlit.app` URL 확인

### 2.3 배포 후 검증

- [ ] URL 접속 시 5탭 정상 로드
- [ ] KPI·차트에 실제 숫자 표시 (mock 아님)
- [ ] 모바일 브라우저에서 탭 가로 스크롤 동작
- [ ] 시트 수정 후 새로고침 시 60초 내 반영
- [ ] `docs/presentation/assets/` 스크린샷 백업 준비

### 2.4 제출 패키지

| 포함 | 제외 |

## 3. 보안

### 3.1 민감정보 분류

| 데이터 | 민감도 | 1차 조치 |
|--------|--------|----------|
| 매출·점유율 | 사업 기밀 | 공개 URL 시 데모 시트·익명 호텔명 검토 |
| 청소·검수 직원명 | 개인정보 가능 | 제출용 실명 최소화 |
| VOC 리뷰 | 샘플(가명) | 실연동 시 비식별 |
| OpenAI | 외부 전송 | Key optional, 기본 규칙 요약 |

### 3.2 기술적 보안

| 항목 | 구현 |
|------|------|
| 시크릿 관리 | `.env` / Cloud Secrets, git 제외 |
| Sheets 접근 | 공개 CSV (인증 없음) — 2차에서 서비스 계정 |
| HTML 렌더링 | `app.py` `_html_esc()` 사용 |
| SSL | `requests` + `certifi` (`lib/sheets.py`) |
| 프록시 우회 | `session.trust_env = False` (로컬 프록시 403 방지) |
| 실패 정책 | mock 대체 금지, 명시적 오류 표시 |

### 3.3 Phase 1 보안 스캔 (권장)

```bash
pip install pip-audit bandit
pip-audit -r requirements.txt
bandit -r app.py lib/ data/ -ll
```

High 이상 취약점 0건 목표.

## 5. 트러블슈팅 (배포 환경)

| 증상 | 원인 | 해결 |
|------|------|------|
| ImportError: altair | requirements 누락 | `pip install -r requirements.txt` 재배포 |
| 실적 시트 로딩 실패 | 시트 비공개·탭명 오류 | 공유 설정·`revenue`/`Rooms` 확인 |
| Proxy 403 (로컬) | HTTP_PROXY 환경변수 | `trust_env=False` (이미 적용) |
| 빈 대시보드 | Secrets 미설정 | Cloud Secrets에 SHEET_ID 확인 |

로컬 실행 상세: [docs/presentation/04_실행방법.md](../presentation/04_실행방법.md)
